# Project context for AI assistants

## Critical convention: the data-access seam

This project is being prepared for migration off Base44. To keep the
migration mechanical, ALL data access goes through `src/lib/data/*`
modules. **Never write `base44.entities.X`, `base44.auth.X`, or
`base44.functions.invoke` from any file outside `src/lib/data/`.**

If you need data access in a feature you're building:
1. Check `src/lib/data/index.js` for the relevant module.
2. If a function you need doesn't exist, add it to the module first.
3. If a whole entity is missing, create it from `src/lib/data/_template.js`.
4. Then use the data layer in your feature code.

An ESLint rule (`no-restricted-syntax` on `base44.*`) enforces this
mechanically — direct calls outside the seam fail the build.

## Other conventions in this project

- All user-facing text input must use the profanity guard
  (`src/lib/useProfanityGuard.js`). See the convention comment at the
  top of that file.
- All data fetched from per-user entities must pass through
  `filterAfterReset` from `src/lib/accountReset.js` — this is a
  defensive layer that hides records predating the user's last
  account reset.
- All UI strings go through `t(...)` from `useLanguage()` and have
  matching keys in all 15 locale blocks across `src/lib/i18n-part1.js`
  (en, es, fr, de, pt, it, ja, ko) and `src/lib/i18n-part2.js`
  (zh, ar, hi, ru, tr, pl, nl).
